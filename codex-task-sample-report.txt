在项目中实现一个新的只读统计页面和后端接口，严禁对 192.168.0.2 上 SQL Server 做任何写操作，只能 SELECT。

需求：
1. 新增页面，标题固定为：本周样衣样板数统计表（实际统计最近六天，含今天）
2. 数据来源：SQL Server 192.168.0.2:1433 数据库 UFDATA_328_2020，账号 Sa 密码 Sa123
3. 只查询这两张对象：v_HY_FZ_Indication、v_HY_FZ_IndicationDetail
4. 分组字段：cDBPersonName（版师）
5. 出版数：按主表 ID 关联明细表 ID，汇总明细 iQuantity
6. 日均出版数：出版数 / 6，保留2位
7. 存版数：按 cSampleStatus 含“存版”统计；若无则为0
8. 平均周期：按明细 dDeliveryDate 与主表 dDate 的日期差平均值（天），保留1位
9. 结果表第一行必须是“技术科”，代表所有版师汇总总计
10. 最后一行必须是“平均值”，代表除“技术科”外各版师行的算术平均
11. 页面列：版师、出版数、日均出版数、存版数、平均周期
12. 页面风格做得整洁一点，适配当前若依项目风格
13. 后端接口必须是只读 GET，不要新增任何写逻辑
14. 尽量沿用项目现有 controller/service/mapper/domain/vue/api 结构
15. 如果需要增加第二数据源，请在项目中最小化改动实现，并确保查询代码不会执行写操作
16. 完成后输出修改了哪些文件、接口路径、前端路由/页面路径，并说明关键 SQL 逻辑

补充说明：
- 你可以在后端中使用 JDBC 直接连 SQL Server，避免大改现有 MySQL 数据源
- 最近六天建议按主表 dDate 筛选，范围为今天及前5天
- cDBPersonName 为空时可显示为“未分配”
- 不要碰无关文件
